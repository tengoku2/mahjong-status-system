# 局単位スタッツ定義

この文書は、TANKI連携で取得する局単位データから算出するスタッツの定義を固定するためのものです。

今後、Discord表示、Web表示、集計API、DB設計はこの定義を基準にします。

## 基本方針

- 分母は、特記がない限り `参加局数` を使う
- `参加局数` は、そのプレイヤーが着席していた局数
- 半荘途中終了や強制終了で未確定の局は、自動集計の母数に含めない
- 途中終了した半荘は、初期方針では自動登録しない
- 3人戦 / 4人戦は必ず分離集計する
- 東風戦 / 半荘戦は将来的に分離可能な形で保存する

## 用語

- `参加局数`
  - 対象プレイヤーが着席していた局数
- `和了局数`
  - 対象プレイヤーが和了した局数
- `放銃局数`
  - 対象プレイヤーがロン放銃した局数
- `副露局数`
  - 対象プレイヤーが1回以上副露した局数
- `立直局数`
  - 対象プレイヤーが立直した局数
- `流局局数`
  - その局自体が流局で終わった局数
- `流局聴牌局数`
  - 流局時に対象プレイヤーが聴牌していた局数
- `ツモ和了局数`
  - 対象プレイヤーがツモ和了した局数
- `ダマ和了局数`
  - 対象プレイヤーが立直なしで和了した局数
- `和了打点合計`
  - 対象プレイヤーの和了点数合計
- `放銃失点合計`
  - 対象プレイヤーの放銃失点合計
- `和了順数合計`
  - 対象プレイヤーが和了した局の巡目合計

## 主要スタッツ

### 和了率

- 定義:
  - `和了局数 / 参加局数`

### 放銃率

- 定義:
  - `放銃局数 / 参加局数`

### 副露率

- 定義:
  - `副露局数 / 参加局数`

### 立直率

- 定義:
  - `立直局数 / 参加局数`

### ダマ率

- 定義:
  - `ダマ和了局数 / 和了局数`
- 備考:
  - 和了が0回のときは `null` 扱い

### 自摸率

- 定義:
  - `ツモ和了局数 / 和了局数`
- 備考:
  - 和了が0回のときは `null` 扱い

### 流局率

- 定義:
  - `流局局数 / 参加局数`

### 流局聴牌率

- 定義:
  - `流局聴牌局数 / 流局参加局数`
- 補足:
  - `流局参加局数` は、対象プレイヤーが参加していた流局局数
- 備考:
  - 流局参加が0回のときは `null` 扱い

### 平均和了

- 定義:
  - `和了打点合計 / 和了局数`
- 備考:
  - 和了が0回のときは `null` 扱い

### 平均放銃

- 定義:
  - `放銃失点合計 / 放銃局数`
- 備考:
  - 放銃が0回のときは `null` 扱い

### 平均和了順数

- 定義:
  - `和了順数合計 / 和了局数`
- 備考:
  - 和了が0回のときは `null` 扱い

### 飛び率

- 定義:
  - `飛び終了回数 / 半荘参加回数`
- 補足:
  - これは局単位ではなく半荘単位スタッツ

## 拡張候補

以下は局データ粒度が十分なら追加する。

### 一発率

- 定義:
  - `一発和了局数 / 立直和了局数`

### 裏ドラ率

- 定義案:
  - `裏ドラ発生和了局数 / 立直和了局数`
- 代替案:
  - `裏ドラ枚数合計 / 立直和了局数`
- 備考:
  - どちらを採用するかは保存粒度に依存する

## Discord と Web の役割分離

### Discord向け

- 和了率
- 放銃率
- 副露率
- 立直率
- 平均和了
- 平均放銃
- 自摸率
- 飛び率

### Web向け

- ダマ率
- 流局聴牌率
- 平均和了順数
- 一発率
- 裏ドラ率
- 直接対決
- 時系列推移
- 条件別フィルタ

## 中断・強制終了の扱い

初期仕様では次を採用する。

- 半荘途中終了
- 退席を含む強制終了
- 局情報が未確定の終了

これらは `自動登録しない`。

代わりに watcher / 取り込み側で中断メッセージを出し、必要なら管理者が手動判断する。

理由:

- 半荘成績と局成績の整合を壊さないため
- 放銃率や和了率などの母数汚染を避けるため
- 意図的終了と通常終了を混ぜないため

## 直接対決の扱い

面白い要素ではあるが、初期の主軸にはしない。

初期に持つなら以下までに留める。

- 対戦相手別半荘数
- 対戦相手別平均順位
- 対戦相手別収支
- 同卓時順位分布

より細かい相手別局分析は後段で追加する。
